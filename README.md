# Portal App Inspector

Portal App Inspector is a Kotlin Multiplatform app-inspection toolkit for debug builds.

The v1 scaffold contains:

- `:lib`: source runtime and protocol.
- `:plugins:files`: official files browsing plugin.
- `:plugins:network`: official Retrofit network traffic plugin.
- `:app`: Compose/Wasm Portal web UI.
- `:demo`: Android demo app.

## Debug Integration

Apps include the runtime and only the plugins they need:

```kotlin
debugImplementation("io.github.portalappinspector:lib:<version>")
debugImplementation("io.github.portalappinspector.plugins:files:<version>")
debugImplementation("io.github.portalappinspector.plugins:network:<version>")
```

In this repo, the demo app uses project dependencies:

```kotlin
debugImplementation(project(":lib"))
debugImplementation(project(":plugins:files"))
debugImplementation(project(":plugins:network"))
```

The Android runtime auto-starts from a ContentProvider, starts a local Ktor server,
and shows a notification titled `Portal App Inspector active`.

The notification copy action copies a URL like:

```text
https://omkar-tenkale.github.io/portal-app-inspector/connect?host=192.168.0.5&port=4896
```

## Protocol

Source exposes:

```text
GET  /portal/health
GET  /portal/manifest
POST /portal/rpc
```

Requests use integer IDs generated by the Portal client and echoed by Source:

```kotlin
data class PortalPluginRequest(
    val id: Int,
    val pluginId: String,
    val payload: JsonObject,
)
```

`ok = false` means a protocol/runtime failure. Plugins may return domain-specific
errors inside `payload` while still returning `ok = true`.

## Build

```bash
./gradlew :lib:build :plugins:files:build
./gradlew :app:wasmJsBrowserDistribution
./gradlew :demo:assembleDebug
```
